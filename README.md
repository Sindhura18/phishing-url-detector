# 🔐 Phishing URL Detector

A machine learning system that classifies URLs as **phishing** or **legitimate** in real time, using a Random Forest classifier trained on the [UCI Phishing Websites Dataset](https://archive.ics.uci.edu/ml/datasets/phishing+websites).

---

## 🎯 Problem

Phishing attacks trick users into clicking malicious links by mimicking trusted websites. Detecting these URLs **before** the user clicks is critical for cybersecurity. This project automates that detection using ML, achieving ~96% accuracy on the UCI benchmark dataset.

---

## 🏗️ Architecture

```
URL Input
    │
    ▼
Feature Extraction (10 URL-based features)
    │
    ▼
Random Forest Classifier (trained on UCI dataset, 11,000 samples)
    │
    ▼
Prediction: Phishing ⚠️  or  Legitimate ✅  + Confidence Score
```

## 🚀 Getting Started

### 1. Clone and install dependencies

```bash
git clone https://github.com/Sindhura18/phishing-url-detector.git
cd phishing-url-detector
pip install -r requirements.txt
```

### 2. Download the dataset

Download the UCI Phishing Websites Dataset from Kaggle:  
👉 https://www.kaggle.com/datasets/eswarchandt/phishing-website-detector

Save it as `data/phishing.csv`.

### 3. Train the model

```bash
python train.py
```

This will:
- Load and inspect the dataset
- Train a Random Forest classifier
- Print evaluation metrics and feature importances
- Save the model to `models/rf_model.pkl`

### 4. Launch the web app

```bash
streamlit run app.py
```

Open your browser to `http://localhost:8501` and paste any URL to test it.

### Why Random Forest over other models?

- Handles mixed feature types (binary flags, numeric counts) without scaling
- Provides **feature importance** — we can explain *why* a URL is flagged
- Resistant to overfitting compared to a single Decision Tree
- Fast inference: predicts in milliseconds, suitable for real-time use

## 🔍 How Features Are Extracted

The model uses 10 URL-based string features that can be computed instantly without any external API calls:

| Feature | Why it matters |
|---------|----------------|
| `has_ip_address` | Phishing sites use IPs to avoid traceable domain registration |
| `has_at_symbol` | `@` tricks browsers — everything before it is ignored |
| `url_length` | Phishing URLs are often very long to hide the real domain |
| `has_https` | Legitimate sites almost always use HTTPS |
| `has_double_slash` | `//` in paths indicates suspicious redirects |
| `dot_count` | More dots = more subdomains (e.g., `pay.pal.secure.evil.com`) |
| `has_hyphen_in_domain` | Mimics brands: `pay-pal-login.com`, `amazon-secure.com` |
| `url_depth` | Deep paths (`/verify/account/confirm`) hide malicious intent |
| `uses_shortening_service` | Shorteners hide the real destination |
| `has_suspicious_keywords` | Words like `verify`, `login`, `confirm` are commonly abused |
